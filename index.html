rpgLab is a web platform based on online forum play-by-post role-playing games - but customized to fit the needs of the game master and the game.


Background Information


  • 1978

    An online text-based role playing game is a role-playing game played online using a solely text-based interface. Online text-based role playing games date back to 1978.

  • 2005

    Some online-text based role playing games are video games, but some are organized and played entirely by humans through text-based communication. Over the years, games have used TELNET, internet forums, IRC, email and social networking websites as their media.

  • 2020

    There are varied genres of online text-based roleplaying, including fantasy, drama, horror, anime, science fiction, and media-based fan role-play. Role-playing games based on popular media (for example, the Harry Potter series) are common, and the players involved tend to overlap with the relevant fandoms.


Play-by-Post Games: Current Status


Rather than following gameplay in real-time, players post messages on such media as bulletin boards, online forums, Chatrooms (such as like AOL, hangouts and Yahoo chat) and mailing lists to which their fellow players will post role- played responses without a real limit or timeframe. Of late such blogging tools and sites as LiveJournal have been utilized for this purpose. This includes such games as play-by- email (or PBEM) RPGs. The origins of this style of role-playing are unknown, but it most likely originated in some form during the mid-to-late 1980s.

Typically, a game master (Admin) is tasked not only with world creation, but trying to set up their unique play-post-game within the bounds of whatever platform they have chosen. This most often happens on forum software like myBB or phpBB -- although alternative platforms, such as Reddit, Wordpress, Facebook, Tumblr, and Skype are also frequently used, due to the knowledge needed to set up and run the aforementioned forum software.

The anatomy of a PBP-RPG dictates that each post comes from a different character. Characters all have attributes such as class (eg, Vampire), personality, history, abilities, and the list can go on. Add to this a universal story setting, world constraints, individual timelines, any character or world events... and you could see how this would quickly get overwhelming to keep track of, for both players and Admin alike.


Thus, the birth of rpgLab. A platform that was created for an Admin to easily set up and customize their very own play-by-post RPG.

Features

Features


Author Account

Users would be able to sign up for a Primary Account, also referred to as an Author account or Main Account.

Character Accounts

Primary users would be able to create secondary/sub-accounts. These would act as character accounts. Primary users would be able to switch between these sub-accounts to facilitate roleplaying under different character aliases.

Customized Structure

Though the platform would be based on discussion forum software, the structure would be customized and geared toward the world scaffold of a typical roleplaying setting. This would include areas, sub-areas, threads and posts.


Custom Profiles

Profiles (also known as Character Sheets) would be tied to each account. These could be expanded upon and easily customized by the admin. This would make keeping track of character attributes much easier.

Random Events

Random Events act like writing prompts for users. These would randomly appear within parameters set by the Admin and show Globally (forum-wide), Locally (area-wide), Sub-locally (sub-area-wide) or at Thread level (thread-wide)

Awards

Admins would be able to give users awards that would appear on their profile page.


Current Events

As various events are always being run in-game by players, a calendar/event system would be beneficial. Events could be inserted, and shown Globally (forum-wide), Locally (area-wide), Sub- locally (sub-area-wide) or at Thread level (thread-wide) in the areas they applied to with specified start/end times.

Notifications

Users would receive notifications based on activity like new messages or new replies to threads.

Inbox System

Users would be able to send, view, delete messages.
